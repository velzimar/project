La combinaison:

Soit E un ensemble fini de cardinal n et p un entier naturel. Les combinaisons de cet ensemble sont ses sous-ensembles (ou ses parties). Une p-combinaison de E (ou p-combinaison sans répétition de E, ou encore combinaison sans répétition de n éléments pris p à p) est une partie à p éléments de E. On utilise la formule suivante : Cpn = n!/((n-p)!p!) 

0)Def fn Comb( n,p : integer ) : integer
1)C := Fact(n)/( Fact(n-p)*Fact(p) )
2)Comb := C
3)Fin Comb

L'arrangement:

Lorsque nous choisissons p objets parmi n objets et que l’ordre dans lequel les objets sont sélectionnés revêt une importance, nous pouvons les représenter par un p-uplet d'éléments distincts et on en constitue une liste ordonnée sans répétition possible, c'est-à-dire dans laquelle l'ordre des éléments est pris en compte (si l'on permute deux éléments de la liste, on a une liste différente, et un élément ne peut être présent qu'une seule fois). On utilise la formule suivante : Apn = n!/((n-p)!) 

0)Def fn Arr( n,p : integer ) : integer
1)A := Fact(n)/Fact(n-p)
2)Arr := A
3)Fin Arr

Relation de réccurance:

En mathématique le nombre de combinaison peut être détèrminer selon la formule suivante: Cnp = Cn-1p-1 + Cn-1p 

0)Def fn Comb_Recursive( n,p : integer ) : integer
1)Si (p = 0) ou (p = n) alors
	Comb_Recursive := 1 
  Sinon		
	Comb_Recursive := Comb_Recursive(n-1,p-1)/Comb_Recursive(n-1,p)
2)Fin Comb_Recursive
